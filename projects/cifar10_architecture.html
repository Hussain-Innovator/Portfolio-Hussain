Enhanced CNN — CIFAR-10
Image Classifier

A custom hybrid CNN architecture combining Residual Connections, Squeeze-and-Excitation Attention, and Depthwise Separable Convolutions — trained on 50,000 images across 10 object categories. Achieves 90.93% test accuracy and 98.71% Top-3 accuracy. Deployed via Streamlit and Hugging Face Spaces using ONNX Runtime.

Author: Hussain Dataset: CIFAR-10 · 60,000 images Framework: TensorFlow 2.15 · Keras Deployment: Streamlit · Hugging Face · ONNX

Project overview

CIFAR-10 is one of the most studied benchmarks in computer vision — 60,000 colour images at 32×32 pixels across 10 mutually exclusive categories. While the dataset is small by modern standards, building a high-performing classifier from scratch without using a large pre-trained model is a meaningful engineering challenge that requires careful architectural decisions.

"Rather than fine-tuning a large pre-trained model, we designed a compact custom architecture from first principles — combining four research techniques to push beyond 90% accuracy while keeping the model lightweight enough to run on CPU."

The architecture builds on ideas from ResNet (residual connections for gradient flow), SENet (channel-wise attention), MobileNets (depthwise separable convolutions for efficiency), and NIN (global average pooling as classifier head). The training pipeline uses MixUp augmentation, label smoothing, and cosine annealing with linear warmup — all contributing measurable accuracy gains over a naive baseline.

Dataset — CIFAR-10

CIFAR-10 was collected by Alex Krizhevsky and Geoffrey Hinton at the University of Toronto. Each image is a 32×32 RGB colour photograph labelled into one of 10 everyday object categories. The dataset is balanced — exactly 6,000 images per class — split into 50,000 training and 10,000 test images.

Data augmentation pipeline

Two complementary augmentation strategies are applied at training time. Standard geometric augmentations run per-image inside the tf.data pipeline. MixUp is applied at the batch level after geometric augmentation.

Standard Geometric Augmentation
Random horizontal flip, pad-and-crop (pad 4px per side with reflection, then random crop back to 32×32), random brightness jitter (±0.1), random saturation jitter (0.8–1.2). Applied per-image in parallel using tf.data AUTOTUNE. Teaches spatial invariance and prevents positional overfitting.
MixUp Augmentation (Zhang et al., ICLR 2018)
Applied at batch level. Pairs of images and their one-hot labels are linearly interpolated: x̃ = λ·xᵢ + (1−λ)·xⱼ and ỹ = λ·yᵢ + (1−λ)·yⱼ where λ ~ Beta(0.4, 0.4). Forces the model to learn smooth decision boundaries and acts as implicit label smoothing. Consistently improves generalisation by 1–2% on CIFAR-10.

Architecture pipeline

Input image
32×32×3 · float32 · normalised [0,1]
Stage 0 — input
Raw CIFAR-10 image tensor
Each image is a 32×32 RGB tensor with 3 channels — 3,072 scalar values per image. Pixel values are normalised from [0,255] integers to [0.0,1.0] float32 by dividing by 255. During training, MixUp may produce interpolated images with non-integer effective labels. The batch tensor entering the network has shape (128, 32, 32, 3).
Stem Conv2D(64)
3×3 · padding=same · BN · ReLU · 32×32×64
Stage 1 — stem
Initial feature extraction
A single 3×3 convolution with 64 filters transforms the 3-channel image into 64 feature maps. Padding=same preserves spatial dimensions at 32×32. No pooling yet — spatial detail is kept intact for Block 1. Batch Normalisation stabilises activations and ReLU introduces non-linearity. Parameters: 3×3×3×64 = 1,728 weights (no bias, BN provides shift). This layer is analogous to the stem in ResNet and EfficientNet.
Block 1 — Residual(64)
Conv(64)+BN+ReLU → Conv(64)+BN → SE → Add → ReLU · Identity skip
Stage 2 — residual block 1
Identity shortcut — no projection needed
The first residual block uses two standard 3×3 convolutions with 64 filters each. Because the input (32×32×64 from the stem) and output (32×32×64) have identical shapes, the skip connection passes the input directly to the Add layer — this is He et al.'s identity shortcut (option A). The SE block recalibrates channel weights between the second conv and the Add. After the Add, ReLU is applied. MaxPool(2×2) halves spatial dims to 16×16, then Dropout(0.25) provides regularisation.
Block 2 — Residual(128)
Conv(128)+BN+ReLU → DepthwiseSep(128)+BN → SE → Add → ReLU · Projection skip
Stage 3 — residual block 2
Projection shortcut — 64→128 channel mismatch
The second block doubles channels to 128. The main branch uses a standard 3×3 conv followed by a Depthwise Separable Convolution — a 3×3 depthwise conv (one filter per channel, spatial mixing only) followed by a 1×1 pointwise conv (channel mixing). This reduces parameters by ~8-9× versus a standard conv. Because the skip path carries 64 channels but the main branch outputs 128, a projection shortcut (1×1 Conv2D(128) + BN, no activation) is applied to match dimensions before the Add. Input: 16×16×64. Output after MaxPool: 8×8×128.
Block 3 — Residual(256)
Conv(256)+BN+ReLU → DepthwiseSep(256)+BN → SE → Add → ReLU · Projection skip
Stage 4 — residual block 3
Projection shortcut — 128→256 channel mismatch
The third block doubles channels again to 256. Same pattern as Block 2: standard 3×3 conv then depthwise separable conv, SE attention, and a 1×1 projection shortcut on the skip path to match 128→256 channels. The SE block here operates on 256 channels with a reduction ratio of 16 (256/16 = 16 bottleneck neurons). After the Add and ReLU, MaxPool(2×2) reduces spatial dims to 4×4, and Dropout(0.25) regularises. Output: 4×4×256.
SE Attention Block
GlobalAvgPool → FC(C/16, ReLU) → FC(C, Sigmoid) → Reshape → Multiply
Stage 5 — SE block (key innovation)
Channel-wise attention mechanism
The SE block (Hu et al., CVPR 2018) is embedded in every residual block. It asks: "which of the C feature channels matter most for this image?" Step 1 — Squeeze: GlobalAveragePooling2D collapses (H,W,C) → (1,1,C), producing a channel descriptor. Step 2 — Excitation: two Dense layers (C→C/16→C) with ReLU and Sigmoid learn per-channel attention weights in [0,1]. Step 3 — Scale: the weights are broadcast and multiplied channel-wise onto the feature map. Channels carrying useful information are amplified; noise channels are suppressed. Adds only ~0.2% parameter overhead.
Global Average Pooling
4×4×256 → 256-dim vector · Dropout(0.4) · Dense(10, Softmax)
Stage 6 — classifier head
Global Average Pooling replaces Flatten + Dense(512)
Instead of Flatten + Dense(512) (which would add ~4M parameters), Global Average Pooling averages each of the 256 feature maps spatially — producing a 256-dimensional vector directly from the 4×4×256 tensor. This is 4×4=16 times fewer values than flattening, with no learnable parameters. Dropout(0.4) then regularises the vector, and a single Dense(10) with Softmax produces the final class probabilities. Label smoothing in the loss function prevents overconfident predictions during training.

Training configuration

Optimiser and Loss
SGD with Nesterov momentum (0.9). Base learning rate = 0.1. Loss: Categorical Cross-Entropy with label smoothing = 0.1. Nesterov momentum applies the gradient at a lookahead position, giving slightly faster convergence than standard SGD. Label smoothing prevents the model from becoming over-confident and improves calibration.
Learning Rate Schedule
Linear warmup for the first 5 epochs: LR ramps from 0 → 0.1 linearly. Epochs 5–100: cosine annealing from 0.1 → 1e-5. Warmup stabilises BatchNorm statistics during the early chaotic phase. Cosine annealing provides a smooth, monotonically decreasing LR that avoids sharp drops. ReduceLROnPlateau provides a backup reduction (factor 0.5, patience 8).
Callbacks
ModelCheckpoint saves best val_accuracy weights to Google Drive automatically. EarlyStopping with patience = 15 restores best weights at the end. CSVLogger records every epoch's metrics to training_log.csv. Best checkpoint reached at epoch 86 with val_accuracy = 90.93%.
Hardware and Framework
Trained on Google Colab T4 GPU (~40 minutes for 100 epochs). Framework: TensorFlow 2.15 / Keras. Batch size: 128. Deployed via ONNX Runtime for CPU inference on Streamlit Cloud and Hugging Face Spaces — no TensorFlow dependency at inference time.

Project timeline

Phase 1 — Architecture Design
Literature review and architecture planning
Reviewed ResNet (He et al., 2016), SENet (Hu et al., 2018), MobileNets (Howard et al., 2017), and MixUp (Zhang et al., 2018). Designed the hybrid architecture combining all four techniques. Identified the projection shortcut requirement mathematically before implementation.
Phase 2 — Implementation
TensorFlow/Keras implementation with full training pipeline
Implemented SE block, residual block with automatic projection shortcut detection (checks in_channels != filters), depthwise separable conv, GAP head. Built MixUp augmentation and cosine annealing LR schedule from scratch. Configured Drive-backed checkpointing.
Phase 3 — Training
100-epoch training run on Colab T4 GPU
Trained for 100 epochs. Best model saved at epoch 86 (val_accuracy = 90.93%). Training and validation curves confirmed healthy learning — no overfitting observed (train/val gap ~3.4%). Top-3 accuracy reached 98.71%.
Phase 4 — Deployment
ONNX conversion and multi-platform deployment
Converted trained model to ONNX format using tf2onnx. Deployed Streamlit app to Streamlit Cloud with Drive-hosted model weights. Deployed Gradio app to Hugging Face Spaces using ONNX Runtime — eliminating TensorFlow dependency at inference time entirely.
